## 先搞清楚：VPS IPv6 是什么

IPv6 是 IP 协议的第六版，用 128 位地址取代了 IPv4 的 32 位地址。简单说就是：地址多到根本用不完。

IPv4 地址长这样：`192.168.1.1`
IPv6 地址长这样：`2001:db8::1`

看起来复杂，但其实对于 VPS 用户来说，更重要的不是记地址，而是这几个实际好处：

**1. 地址成本更低**

IPv4 地址现在在市场上的转让价格已经涨到每个 40–60 美元甚至更高。而 IPv6 地址几乎零成本，大多数 VPS 服务商附赠 /64 或 /48 子网，包含数亿个可用 IP。

**2. 未来网络兼容性**

Google、Cloudflare、各大 CDN 全面支持 IPv6。如果你的服务器只有 IPv4，在某些场景下反而成了"旧设备"。

**3. 端对端连通性更好**

IPv6 不需要 NAT（网络地址转换），每台设备都能拥有真实的全球唯一 IP，连接更直接，延迟理论上更低。

**4. 某些特定用途更灵活**

做代理、邮件服务器、爬虫轮换 IP 等场景，一个 IPv6 /64 子网意味着你可以随时切换地址，不用额外花钱买 IP。

## VPS IPv6 的真实使用场景

理论归理论，来说说实际用的时候会碰到什么情况：

### 场景一：建站 + CDN 加速

现代 CDN（如 Cloudflare）与源站之间支持 IPv6 通信。如果你的 VPS 带 IPv6，Cloudflare 可以通过 IPv6 回源，某些地区的连接效率更高。建站用户配置 A 和 AAAA 记录，双栈访问，兼容性更好。

### 场景二：邮件服务器

搭建自有邮件服务器（如 Mailcow、Mailu）时，IPv6 能让你在 IP 信誉上更灵活。如果某个 IPv4 被某些邮件服务商拉黑，切换到 IPv6 发信是常用的应对手段。

### 场景三：代理 & 流量出口

一个 /64 的 IPv6 子网给你 2^64 个地址可以轮换，用于代理、反爬虫绕过等场景，操作空间远超买几个 IPv4 地址。

### 场景四：IPv6 Only 网络测试

如果你的业务需要支持纯 IPv6 用户，或者你在做 IPv6 相关的开发测试，一台带 IPv6 的 VPS 是最便捷的测试环境。

## 实际配置 VPS IPv6 的基本步骤

买到带 IPv6 的 VPS 之后，很多人不知道怎么用起来。这里给个快速入门：

### 检查 IPv6 是否已启用

bash
ip -6 addr show


如果看到类似 `2001:xxxx:xxxx::1/64` 的地址，说明已经分配。

### 测试 IPv6 连通性

bash
ping6 ipv6.google.com


### 配置 IPv6 默认路由（如果未自动配置）

bash
ip -6 route add default via <网关地址> dev eth0


具体网关地址在 DMIT 后台控制面板里可以查到。

### 配置 /64 子网地址（用于代理轮换等场景）

bash
# 添加一个子网内的新地址
ip -6 addr add 2001:xxxx:xxxx::2/64 dev eth0


配合代理工具使用时，通常在配置文件里指定 `outboundTag` 绑定到不同的本地 IPv6 地址即可。
